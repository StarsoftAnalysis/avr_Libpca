23 /**
24  * @mainpage Introduction & Library overview
25 
26  * The library is at the moment at a very early stage (alpha), but it is functional. The idea behind it is to simplify using Atmega328p peripherals,
27  * by creating a set of simple routines in order to configure and use it quickly. At the moment it has been tested and developed mostly on Arduino UNO rev 3 board, and
28  * for the time being is dedicated to this hardware.The library is written completely in C and is dedicated to speed up everyday C coding for Arduino platform without
29  * having a need to use the Arduino Library, which for the cost of simplifying development process (which is good), separates the user from the hardware quite mostly
30  * (which isn't that good).
31  *
32  * @section features Features
33  *
34  * For the moment the following functionalities are implemented and tested:
35  *
36  *
37  * - Timer Delays (Arduino Library sleep/millis equivalent)
38  * - Timer PWM support
39  * - Beeper - Tone generation (Arduino Library tone equiv.)
40  * - Serial port support - Interrupt driven data reception, data reception by polling, data transmission by polling
41  * - SLIP protocol - serial line data synchronization protocol, to support binary data transfers over serial line
42  * - Simple ADC API
43  * - Simple GPIO abstraction layer
44  * - Software OneWire
45  * - I2C
46  * - SPI
47  *
48  * The library implements basic device drivers for the following devices:
49  * - general 7 segment display driver - driver for LED 7 segment display, implementing interrupt driven multiplexing
50  * - ds1307 - an interface for the RTC clock to map it's internal registers
51  * - ds18b20 - Digital Temperature sensor, the driver is heavilly dependant on the software onewire implementation
52  * - hd44780 - hitachi display driver
53  * - pcd8544 - display driver for the NOKIA 3310 based LCDs
54  *
55  *
56  * @section usage Using the documentation
57  *
58  * For details regarding particular functionality please refer directly to a specific interface file.
59  *
60  * @section examples Examples
61  * Please refer to the provided examples as well for an overview on how to use a particular module.
62  *
63  * @section support Support
64  *
65  * If you are unsure about the something you can also refer to my blog (http://pcarduino.blogspot.co.uk) dedicated to
66  * development in C for arduino platform, where I heavily use this library. You can also contact me through github.com/dagon666
67  *
68  * @section compilation Compiling with your own project
69  *
70  * It is very simple to use this library with your own project. You can customize the config.h settings, pre-build it and then just simply link your project against
71  * the static library. Or you can include it as a compilation target in your Makefile. Below is an example Makefile for a simple project haing only one file: main.c.
72  * The following directory structure has been assumed:
73  *
74  * @verbatim
75  * .
76  * | -- project
77  * | ---- Makefile
78  * | ---- main.c
79  * | -- pca
80  * | ---- ...
81  *
82  * @endverbatim
83  *
84  * @code
85 
86  TARGET=project_executable
87  SOURCES=main.c
88 
89  DEPS=
90  COBJ=$(SOURCES:.c=.o)
91  PCA_PREFIX=../pca
92 
93  CC=avr-gcc
94  OBJC=avr-objcopy
95  MCU=atmega328p
96  CFLAGS=-I. -I$(PCA_PREFIX)/include/ -Wall -Os -DF_CPU=16000000UL -std=c99
97  LDFLAGS=-lpca -L$(PCA_PREFIX)
98 
99  all: $(TARGET)
100 
101  %.o: %.c $(DEPS)
102  @echo -e "\tCC" $<
103  @$(CC) -mmcu=$(MCU) -c -o $@ $< $(CFLAGS)
104 
105  libpca.a:
106  @echo -e "\tBUILDING PURE C ARDUINO LIB"
107  $(MAKE) -C $(PCA_PREFIX)
108 
109  $(TARGET): $(COBJ) libpca.a
110  @echo -e "\tLINKING CC" $<
111  @$(CC) -mmcu=$(MCU) -o $(TARGET) $(COBJ) $(LDFLAGS)
112  $(OBJC) -O ihex -R .eeprom $(TARGET) $(TARGET).hex
113 
114  * @endcode
115  *
116  */
