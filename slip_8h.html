Detailed Description

API to implement a SLIP protocol over serial line.

The reason behind this is to synchronize the receiver and transmitter, so receiver does not loose any data send by the transmitter (start to listen a little after transmitter just sent the data) or doesn't receive any junk before the transmitter started to send the data itself. For more information about SLIP protocol please refer to RFC 1055 http://www.ietf.org/rfc/rfc1055.txt
